post /hardware

Creates a new hardware with the given properties

  • Request
  • Response

Headers

  • Authorization: required(String)

    Basic authentication encoded as a Base64 string, with the credentials in the form 'username:password'

Body

Media type: application/json

Type: any

Example:

{
  "hardwareName": "GTX 960",
  "displayName": "GTX 960",
  "operatingSystem": "WINDOWS",
  "multiplier": 21.33
}

HTTP status code 201

The hardware has been created

Headers

  • location: required(String)

    The URL to the created hardware

Body

Media type: application/json

Type: any

Example:

{
  "id": 1,
  "hardwareName": "GTX 960",
  "displayName": "GTX 960",
  "operatingSystem": "WINDOWS",
  "multiplier": 21.33
}

HTTP status code 400

The given payload is invalid

Body

Media type: application/json

Type: any

Example:

{
  "invalidObject": {
    "id": 0,
    "hardwareName": " ",
    "operatingSystem": "invalid",
    "multiplier": -1.0
  },
  "errors": [
    "Attribute 'hardwareName' must not be empty",
    "Attribute 'displayName' must not be empty",
    "Attribute 'operatingSystem' must be one of: [WINDOWS, LINUX]",
    "Attribute 'multiplier' must be over 0.0"
  ]
}

HTTP status code 401

System user cannot be successfully logged in with provided credentials

HTTP status code 403

System user does not have the correct role to perform this request

HTTP status code 409

A hardware with the same 'hardwareName' already exists

HTTP status code 502

An error occurred connecting to an external system to verify the hardware

HTTP status code 503

The system is not in a valid state to execute write requests
